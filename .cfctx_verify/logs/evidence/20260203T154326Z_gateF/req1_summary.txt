# INITIAL_SETTINGS（初期設定の置き場と運用）

## 目的
- Developer が「エージェント → 役割」を割り当てるための初期設定の置き場を定義する。
- 3ファイル（CLAUDE/AGENTS/GEMINI）は、この初期設定を参照する入口とする（固定ロールにしない）。

## 初期設定ファイル
- 設定例: `.cfctx/agent_role_assignment.example.yaml`
- 実運用の配置候補: `.cfctx/agent_role_assignment.yaml`（名称は運用で決めてよい）

## 参照関係（簡易図）

```
Developer 設定
  └─ .cfctx/agent_role_assignment*.yaml
        └─ CLAUDE.md / AGENTS.md / GEMINI.md
              └─ SSOT: Charter → Mode → Artifacts → Skills
```

## 注意
- Repo Lock（cf-guard.sh）とは別系統の仕組み。混同しない。
- 役割の行動規範は SSOT（Charter → Mode → Artifacts → Skills）に従う。

# agent_role_assignment.example.yaml
# Developer が「エージェント → 役割」を割り当てる例。
# 任意のエージェントが任意の役割を担い得る前提。
# SSOTではなく、運用上の初期設定ファイル（読み込み想定）。

# 例A: 小規模（1人多役）
claude_code:
  roles: [Crafter, CI/QA]

codex:
  roles: [Orchestrator, Architect]

gemini:
  roles: [Auditor]

# 例B: 1エージェントで全役割
single_agent:
  roles: [Architect, Crafter, Orchestrator, CI/QA, Auditor]

# 例C: 分業（役割ごとに担当を分ける）
role_owner_architect:
  roles: [Architect]
role_owner_crafter:
  roles: [Crafter]
role_owner_orchestrator:
  roles: [Orchestrator]
role_owner_ciqa:
  roles: [CI/QA]
role_owner_auditor:
  roles: [Auditor]
